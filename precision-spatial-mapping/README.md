## 🧮 Thin Plate Spline (TPS) Interpolation
A core feature of this skill is transforming coarse satellite grids (e.g., $0.25^\circ$) into high-resolution maps (e.g., 30m) before applying administrative boundaries. 

Instead of standard linear interpolation (which leaves blocky artifacts), this workflow mandates the **Thin Plate Spline (TPS)** method. The TPS algorithm minimizes the following energy function:

$$ E(f) = \sum_{i=1}^N (f(x_i, y_i) - z_i)^2 + \lambda \iint \left( \left(\frac{\partial^2 f}{\partial x^2}\right)^2 + 2\left(\frac{\partial^2 f}{\partial x \partial y}\right)^2 + \left(\frac{\partial^2 f}{\partial y^2}\right)^2 \right) dx dy $$

Where:
- The **first term** ensures the interpolated surface $f(x, y)$ fits the observed coarse data points $z_i$.
- The **second term** represents the "bending energy" of the surface (the penalty for roughness).
- $\lambda$ is the smoothing parameter that balances data fidelity and surface smoothness.

### TPS Code Example
This is a clean, standalone implementation of the TPS logic used in our workflow (independent of any specific environmental or ecological models).

```matlab
% 1. Coarse Grid (Observations)
[lon_coarse, lat_coarse] = meshgrid(linspace(130.0, 131.0, 10), linspace(34.0, 33.0, 10));
data_coarse = peaks(10) + randn(10, 10) * 0.5; % Synthetic data

obs_lon = lon_coarse(:)';
obs_lat = lat_coarse(:)';
obs_val = data_coarse(:)';

% 2. Target High-Resolution Grid
[lon_fine, lat_fine] = meshgrid(linspace(130.0, 131.0, 200), linspace(34.0, 33.0, 200));

% 3. Thin Plate Spline (TPS) via Curve Fitting Toolbox
try
    st = tpaps([obs_lon; obs_lat], obs_val);
    data_fine_flat = fnval(st, [lon_fine(:)'; lat_fine(:)']);
    data_fine = reshape(data_fine_flat, size(lon_fine, 1), size(lon_fine, 2));
catch
    % Fallback to biharmonic spline if toolbox is missing
    data_fine = griddata(obs_lon, obs_lat, obs_val, lon_fine, lat_fine, 'v4');
end
```

## 🛠️ Prerequisites

To successfully execute the scripts generated by this skill, your local environment must meet the following requirements:

### Python Environment
Ensure you have the following libraries installed (`pip install ...` or `conda install ...`):
- `numpy`, `matplotlib`, `netCDF4` (Data handling & plotting)
- `geopandas`, `shapely` (Shapefile parsing and geometry operations)
- `scipy` (Specifically `scipy.interpolate.RBFInterpolator` or `griddata` for cubic interpolation)
- `rasterio` (For high-performance, pixel-perfect geometry masking)

### Matrix/Scientific Computing Environment
If generating scripts for matrix-based scientific computing (like MATLAB), ensure your installation includes:
- **Curve Fitting Utilities**: Required for the `tpaps` (Thin Plate Spline) interpolation algorithm.
- **Mapping/Geospatial Utilities**: Required for geometry reading (`shaperead`) and masking (`inpolygon`).

## 🚀 Usage

Install this skill by loading `SKILL.md` into your agent's context or skills library.

When the agent is asked to plot spatial data or remote sensing maps, it will automatically:
1. **Prompt for Execution Mode**: Ask whether to write Python or Matrix-based code, and whether to run it manually or in the background (to avoid timeout limits on massive interpolations).
2. **Downscale Coarse Grids**: Use Thin Plate Spline (TPS) or Cubic interpolation to upscale raw data to high-resolution grids before masking.
3. **Apply Precision Masking**: Parse Shapefiles accurately, explicitly handling `NaN`-separated multi-part polygons (islands, lakes).
4. **Centroid Fallback**: If the target region is smaller than the raw grid resolution (resulting in a blank mask), it will automatically fallback to capturing the nearest grid point to the region's centroid.
5. **Enforce Aesthetic Standards**: Output at 1000 DPI, correct the map aspect ratio using latitude cosine scaling (`daspect`), and utilize transparent backgrounds without drawing hard Shapefile borders.
